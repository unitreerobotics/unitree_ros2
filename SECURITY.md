УВЕДОМЛЕНИЕ О КРИТИЧЕСКОЙ УЯЗВИМОСТИ (HARDWARE DESTRUCTION)

1. Обнаруженная проблема:
В низкоуровневом SDK (unitree_ros2) отсутствует валидация входных данных для управления приводами. Файл low_level_ctrl.cpp принимает параметры tau (крутящий момент), kp и kd без проверки на физические лимиты оборудования.

2. Вектор атаки:
Из-за отсутствия авторизации в протоколе CycloneDDS, любой узел в локальной сети робота (или через незащищенный Wi-Fi/VPN) может отправить сформированный UDP-пакет напрямую на контроллеры моторов.

3. Последствия (Impact):
Подача некорректных значений на обмотки моторов приводит к мгновенному перегреву и физическому сгоранию электроники и механики роботов серий Go2, B2, H1 и G1.

4. Решение:
Требуется внедрение программных ограничителей (clamping) в SDK и настройка политик безопасности в dds_config.xml.

5. Bug Bounty:
Я обнаружил эту уязвимость и готов предоставить детальный PoC (код для воспроизведения), подтверждающий возможность дистанционного уничтожения оборудования, в обмен на соответствующее вознаграждение.

Контакт: Прошу службу безопасности Unitree связаться для дальнейшего обсуждения.
